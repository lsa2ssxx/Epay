# TokenPay 插件 - Epay 安装说明

插件文件已安装到当前目录，并按官方说明完成了 Epay 源码修改。

## 已完成的步骤

1. **源码修改**：已修改 `admin/ajax_pay.php`，支付方式「调用值」允许下划线（支持如 `USDT_TRC20`、`EVM_ETH_ETH` 等）。
2. **插件与图标**：已复制 `TokenPay_plugin.php` 与 TokenPay 图标到 Epay 对应目录。

## 你需要完成的步骤

### 1. 执行数据库 SQL

在 Epay 使用的数据库中执行：

- **必须**：根据你的数据库表前缀选择对应的 SQL 文件：
  - 表前缀为 `pre_`：执行 `增加字段长度.sql`
  - 表前缀为 `pay_`：执行 `增加字段长度_pay前缀.sql`
  
  此 SQL 用于扩展插件 `types` 字段，否则 TokenPay 多币种无法写入。

### 2. 后台刷新插件列表

登录 Epay 后台 → **支付接口** → **支付插件** → 点击 **刷新插件列表**，确认列表中出现 **TokenPay**。

### 3. 添加支付方式（二选一）

- **方法一**：在后台 **支付接口** → **支付方式** 中，为每个币种手动添加一条支付方式。
- **方法二**：在数据库中执行对应的 SQL 文件，一次性添加所有 TokenPay 支持的币种：
  - 表前缀为 `pre_`：执行 `增加支付方式.sql`
  - 表前缀为 `pay_`：执行 `增加支付方式_pay前缀.sql`

### 4. 添加支付通道

在后台 **支付接口** → **支付通道** 中为每个币种添加通道，配置：

- **API 地址**：TokenPay 服务地址，末尾不要带斜线，如 `https://token-pay.xxx.com`
- **方式**：选择对应币种（如 TRX、USDT_TRC20、EVM_ETH_ETH 等）
- **APP ID**：任意字符
- **API 秘钥**：TokenPay 的 API 密钥

**注意**：请根据你的数据库表前缀选择对应的 SQL 文件。如果表前缀既不是 `pre_` 也不是 `pay_`，请手动修改 SQL 文件中的表名。
